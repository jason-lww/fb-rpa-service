## 1. 功能概述

### 1.1 背景

WA 账号在注册、投养、分配和日常触达过程中依赖稳定的代理 IP。当前简单版 IP 管理已具备 IP 需求推算、IP 列表、新增、编辑、删除和健康指标展示能力，但页面信息组织较基础，缺少面向运营和运维日常判断的完整交互规范。

本需求基于新版交互样式重构 IP 管理页面，要求保留当前简单版所有功能，并强化统计概览、列表指标解释、批量操作、删除限制、健康风险识别和与 Phone 资源的联动关系。

### 1.2 目标

- 让运营/运维在一个页面内快速判断当前 IP 资源是否满足 Instant/Phone 使用需求。
- 通过封号率、负载率、健康度、WA 存活时间、WA 使用时间等指标识别高风险 IP。
- 支持 IP 资源的新增、编辑、单个删除、批量删除，保证已绑定 Phone 的 IP 不被误删。
- 为后续自动分配 IP、自动换绑 IP、异常告警等能力预留数据口径和操作入口。

### 1.3 使用角色

| 角色 | 使用场景 | 权限范围 |
|------|----------|----------|
| 运营主管 | 查看 IP 资源供给是否充足，评估 IP 健康情况，决定是否采购新增 IP | 当前甲方或权限范围内全部 IP |
| 运营专员 | 录入新 IP，维护代理账号和密码，处理低健康度 IP | 当前甲方或被授权机构范围内 IP |
| 运维人员 | 监控 IP 在线状态、负载、掉线、封号等风险，执行换绑前后的资源维护 | 当前甲方或系统授权范围内 IP |
| 超级管理员 | 跨甲方查看和维护 IP 资源 | 全部甲方 IP |

### 1.4 模块入口

| 菜单 | 路由 | 页面标题 |
|------|------|----------|
| WA管理 / IP管理 | `/wa-management/ips` | IP管理 |

## 2. 页面结构

新版 IP 管理页面采用「顶部统计概览 + IP 地址列表」两段式布局。

### 2.1 顶部统计概览区

顶部统计区用于展示 IP 资源需求推算结果，并提供刷新能力。

| 指标 | 示例 | 说明 | 口径 |
|------|------|------|------|
| 总Instant数 | 14 | 当前甲方或权限范围内登记的 Instant 总数 | `count(WaPhone.instantId)` |
| 每IP可用Phone数 | 5 | 单个 IP 可绑定的 Phone 数量上限 | 读取 WA 配置 `phonesPerIp` |
| 需要IP总数 | 3 | 满足当前 Instant 数所需的理论 IP 数 | `ceil(总Instant数 / 每IP可用Phone数)` |
| 当前活跃IP | 4 | 当前在线且可参与分配的 IP 数 | `status = ACTIVE` |
| 需新增IP数 | 0 | 当前资源缺口 | `max(0, 需要IP总数 - 当前活跃IP)` |

**交互要求：**
- 点击「刷新」按钮，重新拉取统计概览和 IP 列表数据。
- 当「需新增IP数」大于 0 时，数字使用风险色突出展示；等于 0 时使用正常色。
- 统计区刷新失败时，保留上一次成功数据，并提示「IP需求数据刷新失败，请稍后重试」。
- 统计区必须保留当前简单版功能，不因新版 UI 调整而删除任何指标。

### 2.2 IP 地址列表区

列表区用于维护 IP 资源和查看 IP 质量指标。

**基础交互：**
- 列表默认按创建时间倒序展示；如存在风险排序策略，可优先展示离线、满载、高封号率、低健康度 IP。
- 表格支持横向滚动，操作列固定在右侧。
- 表格左侧提供复选框，支持批量删除；已关联 Phone 的 IP 不允许勾选。
- 列表右上角提供「新增IP」按钮。
- 当选中至少 1 条可删除 IP 时，展示「批量删除（N）」按钮和「已选择 N 项」提示。

## 3. 列表字段

| 字段 | 类型 | 是否必显 | 说明 |
|------|------|----------|------|
| 复选框 | 操作控件 | 是 | 仅 `关联Phone = 0` 的 IP 可勾选 |
| IP地址 | 文本 | 是 | 代理服务器 IP 地址，如 `103.45.67.12` |
| 端口 | 数字 | 是 | 代理端口，如 `8443` |
| 账号名 | 文本 | 是 | 代理登录账号 |
| 状态 | 标签 | 是 | 在线/离线 |
| 上线时间 | 时间 | 是 | IP 首次上线或最近一次恢复在线时间 |
| 累计服务时间 | 文本 | 是 | 该 IP 自上线以来累计提供代理服务的总时长 |
| 封号率 | 标签 | 是 | 该 IP 下封号 Phone 数 / 总绑定 Phone 数 |
| 负载率 | 进度条 | 是 | 已绑定 Phone 数 / 每 IP 可用 Phone 数 |
| 健康度 | 进度条 | 是 | 0-100 综合评分 |
| WA存活时间 | 文本 | 是 | 该 IP 下号码从激活到封号/下线/当前时间的平均时长 |
| WA使用时间 | 文本 | 是 | 该 IP 下号码被催员实际使用的平均累计时长 |
| 关联Phone | 标签 | 是 | 当前绑定在该 IP 下的 Phone 数 |
| 操作 | 按钮 | 是 | 编辑、删除 |

### 3.1 状态展示规则

| 状态值 | 展示文案 | 样式 | 说明 |
|--------|----------|------|------|
| `ACTIVE` | 在线 | 绿色标签 | IP 可正常代理 WA 账号 |
| `INACTIVE` | 离线 | 灰色标签 | IP 当前不可用，不参与自动分配 |

### 3.2 封号率展示规则

| 条件 | 展示 |
|------|------|
| `banRate = 0` | 绿色标签，展示 `0.0%` |
| `banRate > 0` | 红色标签，展示一位小数百分比 |
| 无绑定 Phone | 展示 `0.0%`，不计入风险 |

封号率字段需提供 tooltip：`该IP下被封号的号码数 / 总绑定号码数；有封号即为红色`。

### 3.3 负载率展示规则

负载率 = `linkedPhoneCount / phonesPerIp`。

| 条件 | 进度条颜色 | 示例 |
|------|------------|------|
| `< 80%` | 绿色 | `3/5` |
| `>= 80% 且 < 100%` | 橙色 | `4/5` |
| `>= 100%` | 红色 | `5/5` |

负载率字段需提供 tooltip：`已绑定Phone数 / 每IP上限数；超过80%变橙色，满载变红`。

### 3.4 健康度展示规则

健康度为 0-100 分，使用进度条展示。

| 分数 | 等级 | 颜色 | 说明 |
|------|------|------|------|
| `80-100` | 健康 | 绿色 | 可正常分配和继续使用 |
| `60-79` | 关注 | 橙色 | 建议关注封号、负载或掉线情况 |
| `0-59` | 风险 | 红色 | 建议暂停新绑定或进行换绑处理 |

健康度字段需提供 tooltip：`综合封号率、负载率、IP状态、在线稳定性的综合评分（0~100分）`。

### 3.5 时间展示规则

| 字段 | 展示规则 |
|------|----------|
| 上线时间 | 使用 `YYYY-MM-DD HH:mm:ss` |
| 累计服务时间 | 小于 24 小时展示 `X小时`，大于等于 24 小时展示 `X天` 或 `X天X小时` |
| WA存活时间 | 有有效统计样本时展示 `X天X小时`，无样本展示 `-` |
| WA使用时间 | 有有效统计样本时展示 `X天X小时`，无样本展示 `-` |

WA 存活时间和 WA 使用时间仅统计满足以下任一条件的 Phone：
- 已封号 Phone。
- 从激活至今或至下线时间已超过 3 天的 Phone。

## 4. 核心操作

### 4.1 新增 IP

**入口：** IP 地址列表右上角「新增IP」按钮。

**弹窗字段：**

| 字段 | 类型 | 必填 | 校验 | 默认值 |
|------|------|------|------|--------|
| IP地址 | 文本输入 | 是 | IPv4 格式；同一甲方下 IP + 端口不允许重复 | 空 |
| 端口 | 数字输入 | 是 | 1-65535 | 8443 |
| 账号 | 文本输入 | 是 | 1-64 字符 | 空 |
| 密码 | 密码输入 | 是 | 1-128 字符，支持显示/隐藏 | 空 |

**交互流程：**
1. 用户点击「新增IP」。
2. 系统打开新增 IP 弹窗，端口默认填充 `8443`。
3. 用户填写 IP 地址、端口、账号、密码。
4. 点击「新增」后进行前端校验。
5. 校验通过后提交；提交期间按钮 loading，禁止重复提交。
6. 提交成功后关闭弹窗，刷新统计概览和列表，提示「新增成功」。
7. 提交失败时弹窗不关闭，提示失败原因。

### 4.2 编辑 IP

**入口：** 列表每行「编辑」按钮。

**可编辑字段：**
- IP地址
- 端口
- 账号
- 密码

**交互要求：**
- 点击「编辑」后打开弹窗，标题为「编辑IP」，回填当前 IP 数据。
- 密码字段默认以密文或占位符展示；如后端不返回明文密码，则保存时未修改密码不得覆盖原密码。
- 点击「保存」后校验字段并提交。
- 保存成功后关闭弹窗，刷新统计概览和列表，提示「更新成功」。
- 已关联 Phone 的 IP 允许编辑账号、密码；修改 IP 地址或端口时需二次确认，提示可能影响已绑定 Phone 的连接稳定性。

### 4.3 删除 IP

**入口：** 列表每行「删除」按钮。

**删除规则：**
- `关联Phone = 0`：允许删除。
- `关联Phone > 0`：禁止删除，提示「该IP下还有 {N} 个Phone关联，请先解绑后再删除」。

**交互流程：**
1. 用户点击「删除」。
2. 系统判断是否有关联 Phone。
3. 如有关联 Phone，阻断删除并提示。
4. 如无关联 Phone，弹出二次确认：`确定删除IP {ipAddress}:{port}？`。
5. 用户确认后提交删除。
6. 删除成功后刷新统计概览和列表，提示「删除成功」。

### 4.4 批量删除 IP

**入口：** 选中一条或多条可删除 IP 后，列表右上角显示「批量删除（N）」。

**规则：**
- 仅允许勾选 `关联Phone = 0` 的 IP。
- `关联Phone > 0` 的 IP 复选框置灰不可选。
- 批量删除前必须二次确认。

**确认文案：**

`确定删除选中的 {N} 个IP地址？此操作不可撤销。`

**结果反馈：**
- 全部成功：提示「已成功删除 {N} 个IP地址」。
- 部分失败：提示「已成功删除 {successCount} 个IP地址，{failCount} 个删除失败，请刷新后重试」。
- 操作完成后清空选中项，并刷新统计概览和列表。

### 4.5 刷新

**入口：** 顶部统计概览区「刷新」按钮。

**刷新范围：**
- 顶部统计概览。
- IP 地址列表。

**交互要求：**
- 刷新期间按钮展示 loading。
- 刷新成功后不额外弹成功提示，避免干扰。
- 刷新失败时展示错误提示，并保留当前页面已有数据。

## 5. 数据口径与业务规则

### 5.1 IP 需求推算

```text
需要IP总数 = ceil(总Instant数 / 每IP可用Phone数)
需新增IP数 = max(0, 需要IP总数 - 当前活跃IP)
```

**补充规则：**
- 总 Instant 数包含待绑定 IP、待激活、已激活、投养中、投养完成、使用中、异常中的所有 Instant。
- 当前活跃 IP 仅统计 `ACTIVE` 状态 IP。
- 离线 IP 不计入可用供给。
- 每 IP 可用 Phone 数从 WA 配置模块读取，不在 IP 管理页面直接修改。

### 5.2 IP 唯一性

同一甲方下，`IP地址 + 端口` 必须唯一。

| 场景 | 规则 |
|------|------|
| 新增重复 IP + 端口 | 阻断提交，提示「该IP和端口已存在」 |
| 编辑为重复 IP + 端口 | 阻断提交，提示「该IP和端口已存在」 |
| IP 相同但端口不同 | 允许，但需记录为不同代理资源 |

### 5.3 健康度评分建议

健康度由后端统一计算并返回，前端仅负责展示。建议初始计算规则如下：

| 因素 | 影响 |
|------|------|
| IP 离线 | 大幅扣分 |
| 封号率升高 | 按比例扣分 |
| 负载率过高 | 满载或接近满载扣分 |
| 累计服务时间较长且稳定 | 适度加分 |
| 关联 Phone 掉线数量 | 按数量扣分 |

健康度评分范围必须限制在 `0-100`。

### 5.4 删除保护

为了避免影响 WA 账号运行，任何已绑定 Phone 的 IP 不允许删除。删除前后必须保持 Phone 与 IP 绑定关系一致，禁止出现 Phone 仍指向已删除 IP 的脏数据。

### 5.5 密码安全

- IP 代理密码属于敏感信息，列表中不展示密码。
- 编辑弹窗中如需展示密码，默认隐藏。
- 后端存储需加密或使用安全凭据存储方案。
- 操作日志不得记录明文密码。

## 7. 接口需求

### 7.1 获取 IP 需求推算

| 项 | 内容 |
|----|------|
| 接口 | `GET /api/v1/wa-management/ips/demand` |
| 入参 | `tenantId` 可由全局甲方上下文传入 |
| 返回 | `IpDemand` |
| 说明 | 用于顶部统计概览区 |

### 7.2 获取 IP 列表

| 项 | 内容 |
|----|------|
| 接口 | `GET /api/v1/wa-management/ips` |
| 入参 | `tenantId`、分页参数、可选筛选参数 |
| 返回 | `WaIp[]` 或分页结构 |
| 说明 | 返回健康度、负载率、封号率等计算后字段 |

### 7.3 新增 IP

| 项 | 内容 |
|----|------|
| 接口 | `POST /api/v1/wa-management/ips` |
| 入参 | `ipAddress`、`port`、`accountName`、`password` |
| 返回 | 新增后的 `WaIp` |
| 说明 | 后端需校验 IP + 端口唯一性 |

### 7.4 编辑 IP

| 项 | 内容 |
|----|------|
| 接口 | `PUT /api/v1/wa-management/ips/{id}` |
| 入参 | `ipAddress`、`port`、`accountName`、`password` |
| 返回 | 成功/失败 |
| 说明 | 未修改密码时不得覆盖原密码 |

### 7.5 删除 IP

| 项 | 内容 |
|----|------|
| 接口 | `DELETE /api/v1/wa-management/ips/{id}` |
| 入参 | `id` |
| 返回 | 成功/失败 |
| 说明 | 后端必须校验无关联 Phone 后才允许删除 |

### 7.6 批量删除 IP

| 项 | 内容 |
|----|------|
| 接口 | `POST /api/v1/wa-management/ips/batch-delete` |
| 入参 | `ids: number[]` |
| 返回 | `successIds`、`failedItems` |
| 说明 | 允许部分成功；失败项需返回原因 |

## 9. 异常与空状态

| 场景 | 页面表现 |
|------|----------|
| IP 列表为空 | 展示空状态文案「暂无IP地址，请先新增IP」 |
| 统计数据加载失败 | 保留旧数据，提示「IP需求数据刷新失败，请稍后重试」 |
| 列表加载失败 | 表格展示错误提示，支持点击刷新重试 |
| 新增重复 IP | 提示「该IP和端口已存在」 |
| IP 格式错误 | 提示「IP格式不正确」 |
| 端口超出范围 | 提示「端口范围为 1-65535」 |
| 删除有关联 Phone 的 IP | 提示「该IP下还有 {N} 个Phone关联，请先解绑后再删除」 |
| 批量删除部分失败 | 展示成功数量和失败数量 |

## 11. 非功能性要求

- 页面首屏加载时间：正常网络下不超过 2 秒。
- IP 列表接口响应时间：P95 不超过 1 秒。
- 支持至少 1,000 条 IP 数据分页查询；如前端暂不分页，需保证 200 条以内渲染流畅。
- 所有接口必须鉴权，并按甲方/角色控制数据范围。
- 密码字段传输必须使用 HTTPS，后端不得明文落库。
- 删除、编辑等写操作必须具备幂等保护和后端数据校验。

---

## 12. 验收标准

- 页面按新版交互展示顶部统计概览和 IP 地址列表，并保留当前简单版全部功能。
- 顶部统计能正确展示总 Instant 数、每 IP 可用 Phone 数、需要 IP 总数、当前活跃 IP、需新增 IP 数。
- 列表能正确展示 IP 地址、端口、账号名、状态、上线时间、累计服务时间、封号率、负载率、健康度、WA 存活时间、WA 使用时间、关联 Phone 和操作。
- 新增、编辑、单个删除、批量删除均可正常操作，并在成功后刷新统计概览和列表。
- 已关联 Phone 的 IP 不允许删除，也不允许在批量删除中被勾选。
- 封号率、负载率、健康度颜色规则符合 PRD 要求。
- IP 地址、端口、账号、密码的必填和格式校验生效。
- 密码不在列表展示，操作日志不记录明文密码。
